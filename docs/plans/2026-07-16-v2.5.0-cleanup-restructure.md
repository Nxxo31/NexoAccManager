# NexoAccManager v2.5.0 — SPEC: Limpieza y Reestructuración Completa

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Eliminar todo código legacy de v2.2/v2.3, dejar un solo proyecto coherente v2.5.0 con arquitectura single-view, tests que validen el estado real del UI, y build NSIS actualizado.

**Architecture:** NexoAccManager es un Electron + React + TS + Zustand app. Vista única sin routing. Tres capas: Header → AccountTable → Dock. Modales overlay. 100% local, AES-256-GCM, contextBridge IPC.

**Tech Stack:** Electron 28, React 19, TypeScript 5, Zustand, framer-motion, Tailwind CSS, vitest, Playwright, axe-core

## Non-Functional Requirements
- **Seguridad**: contextIsolation: true, nodeIntegration: false, sandbox: true — SIEMPRE
- **Cookies**: .ROBLOSECURITY nunca sale del PC
- **IPC**: invoke/handle only, namespacing obligatorio
- **a11y**: WCAG 2.1 AA mínimo, focus-trap en todos los modales, ARIA labels en icon buttons
- **Build**: AppImage + NSIS funcionando, sin warnings críticos
- **Tests**: Unitarios (vitest) + E2E browser (Playwright) + a11y (axe-core) + visual regression

- E2E browser: ~10-12

- Visual regression: ~5-6
